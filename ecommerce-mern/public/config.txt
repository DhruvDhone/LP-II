STEP 22: Configure Nginx

sudo rm -f /etc/nginx/sites-enabled/default

STEP 23: Create Nginx reverse proxy config

sudo tee /etc/nginx/sites-available/ecommerce > /dev/null <<'EOF'
server {
    listen 80;
    server_name _;

    location / {
        proxy_pass http://127.0.0.1:5000;
        proxy_http_version 1.1;

        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;

        proxy_cache_bypass $http_upgrade;
    }
}
EOF

PROJECT EXPLANATION FOR VIVA

This is a full stack e-commerce web application deployed on an AWS EC2 Ubuntu virtual machine.

The frontend is made using HTML, CSS, and JavaScript.
Users can browse products, add products to cart, enter customer details, and place an order.

The backend is made using Node.js and Express.js.
It provides REST APIs for products and orders.

MongoDB is used as the database.
It stores product details and customer orders.
MongoDB is running inside a Docker container on the EC2 instance.

PM2 is used to keep the Node.js app running in the background.

Nginx is configured as a reverse proxy.
Nginx listens on port 80 and forwards requests to Node.js running on port 5000.

Main API endpoints:

GET /api/products
Returns all available products.

POST /api/orders
Stores a new customer order.

GET /api/orders
Returns recent customer orders.

Final result:
The e-commerce website is publicly accessible using the AWS EC2 public IP address:

http://54.221.71.99
